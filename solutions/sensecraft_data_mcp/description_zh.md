## 这个能力做什么

让任意兼容 MCP 的语音助手或 LLM 用自然语言实时访问你的 SenseCAP PaaS 设备数据——注册设备、查询设备 key，或者直接问一句「我的传感器现在读数是多少」，不用打开 App，不用写查询语句。

## 部署后你会得到什么

- 一个设备总览工具：问一句"我的设备怎么样了"就有答案——有几台设备、谁离线、谁电量低，异常优先播报，设备都正常的话也不会啰嗦一大堆。
- 一个单设备详细读数工具：报设备名，把这个设备当前所有通道的读数（温度、湿度、土壤墒情、风速风向、降雨量等等）念成一段人话，不是甩一堆原始数字。
- 一个设备注册工具：报设备名，自动注册并返回它的 EUI。
- 一个设备密钥查询工具：按设备名或 EUI 查询 `device_key`/token，配置固件联网时会用到。
- 一个实时数据工具：按设备名或 EUI 查询该设备**每个通道**各自最新的一条遥测数据，不是只有一个数值。
- 一个历史数据工具：按时间范围拉取历史遥测数据，配套一个用于画图的聚合工具。
- 一个代码模板工具：浏览/读取用于向 SenseCAP 上报传感器数据和设备状态的 Arduino/PlatformIO 代码模板。
- 设备总览和单设备读数这两个工具会直接播报一段基于真实数据组织好的人话；实时数据/历史数据/画图/密钥这几个工具返回的是简短摘要加完整结构化数据，更适合脚本或后续处理，不建议原样念给用户听。

## 适合接到哪些系统

- **小智（XiaoZhi）语音助手** —— 主要场景：接上你的小智 MCP 接入点，开口问设备情况即可。
- **任意其他兼容 MCP 的宿主** —— Claude Desktop、Cursor，或任何会说 Model Context Protocol 的自定义 LLM Agent。

## 给程序对接的接口

| 接口类型 | 说明 | 怎么连接 | 数据格式 |
|---|---|---|---|
| MCP over WebSocket | 出站连接到 MCP 接入点，暴露 9 个工具 | 把 `MCP_ENDPOINT` 设为你的接入点地址 | JSON-RPC（MCP） |

> 这个容器只会主动"出站连接"，不监听任何端口——不需要开放端口，也不用担心防火墙问题。

## 使用须知

- 需要一个已有 Access ID/Key 的 SenseCAP PaaS 账号（sensecap.seeed.cc → API 密钥），并且设备已经在向平台上报数据。
- 如果要接入小智语音助手，需要从小智智控台获取 MCP 接入点地址。
- 凭据只在部署时填入，不会打进 Docker 镜像——之后随时可以用新凭据重新部署这个套餐来更新。
