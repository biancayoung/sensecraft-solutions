# reCamera 实时二维码识别

该方案可以在 reCamera 上直接运行二维码识别。

设备通过 RTSP 输出实时摄像头视频流，并通过 HTTP API 提供最新一次二维码识别结果。SenseCraft Solution 会在预览窗口中显示实时画面，并把二维码内容、检测时间和边框直接叠加在画面上。

## 主要特性

- reCamera 输出 RTSP 实时视频流
- 二维码识别在 reCamera 本地运行
- 独立二维码检测线程，不阻塞视频推流
- 长度为 1 的最新帧队列，避免检测任务堆积
- 通过 HTTP API 获取最新二维码识别结果
- 软件内预览窗口叠加显示二维码识别结果

## 数据链路

| 数据类型 | 协议 | 地址 | 用途 |
| --- | --- | --- | --- |
| 视频流 | RTSP | `rtsp://<device-ip>:8554/live0` | 显示实时摄像头画面 |
| 二维码结果 | HTTP API | `http://<device-ip>:8080/api/qr/latest` | 获取最新一次二维码识别结果 |
| 健康检查 | HTTP API | `http://<device-ip>:8080/api/health` | 检查服务状态 |

## 工作原理

程序在 reCamera 上使用两路视频通道。一路视频通道编码为 H.264，并通过 RTSP 输出。另一路视频通道输出低分辨率 NV21 图像帧，用于二维码识别。

二维码检测线程从长度为 1 的最新帧队列中获取图像。如果检测速度低于摄像头帧率，旧帧会被新帧覆盖。这样可以保证视频流稳定输出，并避免二维码检测延迟不断累积。
